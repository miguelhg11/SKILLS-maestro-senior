# Matriz de Selección de Skills: Maestro Orchestrator (v5.0)

Esta matriz guía la selección de habilidades para garantizar una orquestación de **CALIDAD SUPREMA** y latencia mínima.

## 1. Niveles de Criticidad

| Nivel | Categoría | Descripción | Cuándo Usar |
| :--- | :--- | :--- | :--- |
| **L1** | **CORE** | Fundamentos de orquestación, seguridad y logging. | Siempre (Pre-flight). |
| **L2** | **SOLUTIONS** | Patrones de arquitectura y playbooks probados. | Antes de proponer soluciones. |
| **L3** | **TOOLS** | Herramientas específicas (Frontend, Backend, DevOps). | Durante la implementación. |
| **L4** | **SPECIALIZED**| IA avanzada, Video, Investigación masiva. | Peticiones puntuales. |

## 2. Matriz de Decisión Rápida

| Tarea | Skill Primaria (L1/L2) | Skill de Apoyo (L3/L4) | Protocolo |
| :--- | :--- | :--- | :--- |
| **Nueva App** | `orchestrating-agents` | `idea-to-architecture.md` | Pre-flight + Bootstrap |
| **Seguridad** | `architecting-security` | `configuring-firewalls` | Hard Rules Audit |
| **Investigación** | `notebooklm-research` | `deep-research-method` | Gateway mandatory |

| **DevOps / CI** | `building-ci-pipelines` | `deployment-strategies` | Secure Go-Live |

## 3. Reglas de Selección Senior
1. **Navaja de Ockham**: Si una skill L2 cubre el 80% de la tarea, no invoques tres L3.
2. **Pre-flight Check**: Nunca uses una skill L3/L4 sin haber verificado las reglas L1 (Seguridad) primero.
3. **Audit de Regreso**: Si delegas a OpenCode (L4), la validación final **DEBE** ser realizada por una skill CORE (L1).
